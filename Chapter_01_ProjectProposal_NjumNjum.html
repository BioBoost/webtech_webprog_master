Project Proposal

"Njum Njum"

labict.be

Project description

The client (mister Mustachio Kwestion) wants a web application developed to arrange the meetings with his colleagues for lunch.

Every colleague should be able to create a proposal for a meeting (this includes a date, time, place, description, remark, ...).

If two proposals were to collide in a given timeframe (for example one week (configurable)) then a voting has to take place in which all colleagues can participate. Once decided which meeting wins, for example count the votes 2 days before the meeting (configurable), that meeting is scheduled.

Project description

Colleagues can indicate if they will or will not attend a meeting. Of course they can later still change their mind.

Once a colleague has indicated he will attend the meeting, an Outlook calendar meeting request should be sent to the user.

The day of the meeting (configurable) all attendees should receive an e-mail to remind them of the meeting.

The day before the meeting (configurable) all colleagues that have not yet decided if they will attend the meeting will be send an e-mail.

Project description

Colleagues should be able to register themselves by supplying some basic information (name, e-mail, department, username, password, ...). After this an activation link should be sent to the given mail address to enable the account for login.

Users should be able to configure if they want to receive e-mails for meetings.

Every user should be able to change his/her personal information.

User roles have to be included in order to distinguish the normal users from the administrative users.

Project description

People that register themselves are always given the normal role. It is the job of an administrator to make another user an administrative user.

A history page of past meetings should be available to administrative users.

Project requirements

    The application should be:
  • safe and secure.
  • user friendly and intuitive.
  • thoroughly tested.

Project essentials

Build the web application using state-of-the-art technology.

    It is mandatory to:
  • Use an MVC framework
  • Use a CSS framework
  • Use Ajax (Asynchronous JavaScript and Xml)
  • Use a database
  • Make use of a REST API (for example send tweet when a meeting is scheduled)

Project optionals

    You can optionally:
  • Make use of a JavaScript framework
  • Include Google analytics
  • Supply a REST interface
  • Make use of Test Driven Development (TDD)
